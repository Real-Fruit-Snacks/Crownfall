## What is it?

Crownfall is a top-down medieval **incremental wave-defense game**. You command a row of heroes on a stone wall while progressively tougher waves attack from the field on the right. Every death is meaningful — you bank **Research Points** and **Crowns** to permanently upgrade your fortress, unlock new heroes, elements, skills, hazards, and skins. Hundreds of runs of progression by design.

Built as a single self-contained `index.html` — **no build step, no dependencies, no installs.** Just open the file or visit the GitHub Pages link.

## Features

### Combat
- **6 hero classes** — Archer, Crossbow, Cannoneer, Mage, Cleric, Bard
- **6 enemy archetypes** with matching boss variants — Goblins, Chargers, Shieldbearers, Archers, Healers, Witches
- **8 weapon imbuements** with curated per-hero pools — Fire, Ice, Poison, Bleed, Knockback, Stun, Holy, Mending, Hex
- **6 active skills** with cinematic effects — Smite, Meteor, Heal Wall, Rally, Frost Nova, Stun Bomb
- **5 field hazards** — Wooden Spikes, Caltrops, Tar Pits, Watchfire Braziers, Bear Traps

### Progression
- **Research Book** with 10 themed chapters and 100+ nodes (RP and Crown costs)
- **Crowns** — special boss-drop currency for late-game power spikes
- **5 wall material tiers** — Stone → Reinforced → Iron → Steel → Obsidian
- **Persistent Commanders** that automate hero buying, upgrading, and imbuing per slot
- **Starting-wave ladder** — skip past low-level grind once you've earned it
- **15 achievements** with a "Hall of Records" stats page

### Systems
- **Special event waves** — Blood Moon, Night of the Restless, Goblin Horde, Champion's Challenge, Plague Wave
- **Mid-run random events** — Wandering Merchant, Dark Pact, Royal Decree, Hermit's Blessing
- **12 hero skin variants** unlocked with Crowns
- **Wall visually deteriorates** as HP drops (cracks, broken merlons, side chunks, smoke)
- **Pure synth audio** — every SFX and the looping medieval ambient music is generated by Web Audio API

## Tech Stack

- **HTML5 Canvas 2D** for rendering with `image-rendering: pixelated`
- **Vanilla JavaScript** — no frameworks, no bundler, no transpiler
- **Web Audio API** — all sounds are synth-generated (no asset files)
- **localStorage** for persistent saves (research, crowns, stats, achievements, commanders, settings)
- **CSS-only chrome** — chunky pixel-art borders via `box-shadow inset`

The entire game lives in a single `index.html` (~7,500 lines including all CSS and JS).

## Controls

| Input | Action |
|---|---|
| **Click** | Buy slot, hire hero, collect coin, tap-damage enemy |
| **1–4** | Cast / arm skill in slot N |
| **Q** | End run (banks RP, opens research) |
| **R** | Toggle research book |
| **M** | Mute sound |
| **Space** | Pause / resume |
| **Esc** | Close topmost menu / unarm skill / dismiss popup |

## Running Locally

The simplest option:

```bash
# Just open the file
open index.html      # macOS
start index.html     # Windows
xdg-open index.html  # Linux
```

Or run a tiny static server (recommended for local audio + cleaner cache behavior):

```bash
python -m http.server 5555
# then visit http://localhost:5555
```

No `npm install`, no build pipeline.

## Deploying to GitHub Pages

This repo is already configured for GitHub Pages:

1. Push to GitHub.
2. Repo **Settings → Pages**.
3. Source: **Deploy from a branch** → branch: `main` → folder: `/ (root)`.
4. Save. Your game lands at `https://Real-Fruit-Snacks.github.io/REPO_NAME/` within a minute.

The `.nojekyll` file at the root tells Pages not to process the site through Jekyll, so any files starting with `_` work normally.
